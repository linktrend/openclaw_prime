DAILY USE (no new setup code)
------------------------------------------------------------
After the first successful pairing, reconnect is sticky. You do NOT need a new
setup code for normal use (app restart, phone reboot, leaving the house).

1. Tailscale on iPhone = Connected (same tailnet as the Mini).
2. Mac Mini awake; Lisa gateway LaunchAgent running.
3. Open OpenClaw → Settings → Gateway.
4. Select / Reconnect the paired Tailscale gateway:
     linktrend-mini.tailf7e13a.ts.net:8443  (TLS / Secure ON)
     Full URL if the app asks: wss://linktrend-mini.tailf7e13a.ts.net:8443
5. Chat with Lisa (main). "Local Coder" in the agent list is normal — see note.

FIX “Gateway error… bad response from the server” (wrong port)
------------------------------------------------------------
That error means the phone is still aiming at :443 (Funnel), which is not OpenClaw WS.

1. Settings → Gateway → Forget any entry that is:
     - linktrend-mini… without :8443 (or port 443)
     - macmini.local:18790
2. Add / reconnect with Manual Gateway (if the app offers it):
     Host / URL: wss://linktrend-mini.tailf7e13a.ts.net:8443
     Port: 8443   (if Host and Port are separate fields)
     Secure / TLS: ON
     Token: see OPENCLAW_GATEWAY_TOKEN below (only if the UI has a token field)
3. If Connect asks for pairing after the URL change: on Mac Mini run
     Lisa-Approve-Device.command → type iPhone
   or double-click Lisa-Make-iPhone-Setup.command for a fresh setup code + watcher.
