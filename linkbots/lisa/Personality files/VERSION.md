Motivation: a token review found `TOOLS.md` (always injected, every turn) had absorbed large blocks of situational reference material — Google Workspace (`gws`) detail and the full Cursor/ACP procedure — defeating the point of progressive disclosure for the two files that never got the memo. - **New on-demand files:** `tools/gws.md` (full Google Workspace CLI reference: identity model, scopes, commands, security boundaries, the verified Google Keep limitation and investigation) and `tools/cursor-acp.md` (full Cursor/ACP mechanics: trigger phrases, bind mode, honest-reporting rule, headless CLI quirks). - **`TOOLS.md` trimmed:** ~17.4k → ~7.1k characters. The removed content wasn't deleted — it moved to the two files above, with short summary + pointer left behind.
- **`AGENTS.md`:** LiNKdeveloper Executive Supervision section trimmed to a summary + pointer (the full procedure was already duplicated, in more detail, in `memory/linkdeveloper.md` — no information was lost). Progressive Disclosure index updated with the new `tools/` files and a `memory/INDEX.md` pointer. Added a rule: for files with `sections:` frontmatter, `Grep` the heading and read just that range instead of the whole file.
- **New `memory/INDEX.md`:** one line per file/folder in `memory/`, with a maintenance rule to keep it current as new memory files are added.
- **`sections:` frontmatter added** to `memory/linkdeveloper.md` and `memory/linkdeveloper-alignment.md` — the two largest, most-used on-demand files — so Lisa can jump to a specific heading instead of reading the whole file.
- **Net effect:** the fixed token floor paid on every single request (bootstrap files + tool schemas + system framing) drops from roughly 14,000 tokens to roughly 10,500–11,500 tokens — about a 20–25% reduction — with zero information loss. Everything that was in `TOOLS.md` still exists; it's just loaded only when actually relevant.
- **Verification after deploy:** confirmed via `/context list` and a fresh-session gateway log check (see `PERSONALITY_WORKFLOW.md` § Token Budget for before/after numbers and the exact smoke tests run).

**v1.3 addendum 4 (2026-07-20):** Investigated the local-model coding-delegation gap Carlos flagged (asked Lisa to use a local model for a task; she coded it herself on her own cloud model instead). Findings, confirmed live rather than guessed: (1) the delegation mechanism genuinely exists and works — `sessions_spawn` with `runtime: "subagent"` accepts a `model` override (e.g. `model: "ollama/qwen2.5-coder:7b"`), tested live, spawn returned `resolvedModel`/`modelApplied` confirming it actually ran on that model; (2) but `AGENTS.md`/`TOOLS.md` never told Lisa to use it for this case — the only documented local-coding pattern was Lisa switching her _own_ session model and coding herself (the 7pm–7am/Cursor-unavailable auto-fallback), with no rule at all for "Carlos names a specific model for a task right now"; (3) a real, separate config bug made the natural fallback (Lisa switching her own session to `coder`) actively fail rather than degrade gracefully — `ollama/qwen2.5-coder:7b` (the documented `coder` alias, actually pulled and available in Ollama) was missing from `agents.defaults.modelPolicy.allow`, so a direct session model-switch to it was rejected with `model not allowed: ollama/qwen2.5-coder:7b`; `sessions_spawn`'s subagent `model` override bypasses that same policy check entirely (an inconsistency in the fork worth flagging separately, not fixed tonight). This plausibly explains the original incident end-to-end: the natural move (switch to `coder`) was silently unavailable, and nothing told her to spawn a subagent with the model override instead, so she defaulted to coding it herself. Fixes applied: added `ollama/qwen2.5-coder:7b` to `modelPolicy.allow` (both live and doc-source `openclaw.json`) so the documented `coder` alias actually works; added an explicit routing rule to `AGENTS.md` § Determinism & Model Routing and `TOOLS.md` § Routing rules (new rule 9) — when Carlos names a model for a coding task, delegate via `sessions_spawn`/`model` override and verify `resolvedModel`/`modelApplied`, never silently substitute her own active model.
